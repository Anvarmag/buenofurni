# Руководство по локальной разработке (Local Development) — BUENOFURNI

## 1. Требования к окружению (Node.js Requirements)
Для корректного запуска проектов (Frontend и Server) необходима среда выполнения Node.js.
*   **Рекомендуемая версия:** Node.js v18.x LTS (или выше).
*   **Как проверить версию:** Откройте терминал и выполните команду: `node -v`
*   **Установка:** Если Node.js не установлен или версия ниже 18, скачайте актуальный релиз LTS с официального сайта [nodejs.org](https://nodejs.org/).

## 2. Установка зависимостей (Installing Dependencies)
Проект разделен на две части: клиентскую (`/frontend` — Next.js App Router, Tailwind, TypeScript) и серверную (`/server` — Node.js API для сбора лидов). Зависимости устанавливаются в каждой директории отдельно.

### Для Frontend:
Откройте терминал в корневой папке проекта и выполните команды:
```bash
cd frontend
npm install
```

### Для Server:
Откройте вторую вкладку терминала в корневой папке проекта и выполните:
```bash
cd server
npm install
```

## 3. Запуск режима разработки (Running Development Mode)

### Запуск Frontend
В терминале папки `frontend` выполните:
```bash
npm run dev
```
*   **Порт по умолчанию:** Запускается на `http://localhost:3000`.
*   *Что делать, если порт 3000 занят?* Next.js автоматически предложит использовать следующий свободный порт (например, `3001` или `3002`), либо вам нужно убить процесс, занимающий порт 3000 в диспетчере задач.

### Запуск Server
Во второй вкладке терминала (в папке `server`) выполните:
```bash
npm run dev
# или npm run start (если настроено в package.json)
```
*   **Порт по умолчанию:** Сервер API обычно запускается на порту `3001` (или другом, указанном в `.env`). Отвечает за прием заявок по адресу `http://localhost:3001/api/leads`.

## 4. Настройка переменных окружения (.env Setup)
Для работы сервера необходим файл конфигурации `.env`. Скопируйте шаблон `.env.example` в новый файл `.env` внутри папки `/server`. 

**Описание ключевых переменных:**
*   `PORT` — на каком порту запускать сервер (по умолчанию 3001).
*   `TELEGRAM_BOT_TOKEN` — уникальный токен бота, полученный от @BotFather в Telegram.
*   `TELEGRAM_CHAT_ID` — ID чата или группы, куда бот присылает уведомления о заявках B2C и B2B.
*   `EMAIL settings` (Настройки SMTP):
    *   `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS` — доступы к вашему почтовому серверу (например, Яндекс или Mail).
    *   `EMAIL_FROM` — ящик, от имени которого отправляются письма (часто совпадает с `SMTP_USER`).
    *   `EMAIL_TO` — почта менеджера, куда приходят заявки.

*(Во фронтенде может потребоваться `NEXT_PUBLIC_API_URL=http://localhost:3001`, чтобы формы отправляли данные на нужный локальный адрес вместо продакшена).*

## 5. Рекомендуемый рабочий процесс (Recommended Dev Workflow)
Разработка ведется в двух терминалах одновременно.
1.  **Терминал 1:** Открыт в папке `/server`, работает `npm run dev`. Берет на себя обработку API.
2.  **Терминал 2:** Открыт в папке `/frontend`, работает `npm run dev`. Отрисовывает интерфейс и компоненты.
При нажатии кнопки "Оставить заявку" на фронтенде (`localhost:3000`), запрос успешно уйдет на локальный API сервер (`localhost:3001/api/leads`).

## 6. Частые проблемы (Common Problems)

*   **"package.json not found":** Вы пытаетесь выполнить команду `npm` в корневой папке (`buenofurni`), а не в подпапках. Сначала выполните `cd frontend` или `cd server`.
*   **Port already in use (Порт занят):** Остановите предыдущие запущенные серверы (`Ctrl + C` в терминале) или найдите и завершите процесс, занимающий порт 3000/3001.
*   **Node version mismatch (Версия не совпадает):** Обычно проявляется в виде ошибки `fetch is not defined`. Обновите Node.js до актуальной LTS-версии (18+).
*   **Module not found:** Вы забыли выполнить `npm install` перед запуском проекта. Убедитесь, что папка `node_modules` существует в рабочей директории.
*   **Restarting dev server (Перезапуск сервера):** Если вы изменили глобальные конфиги (например, файлы `.env` или конфиг Tailwind), необходимо остановить сервер комбинацией клавиш `Ctrl + C` и запустить команду `npm run dev` заново.

## 7. Просмотр сборки для продакшена (Production Preview Mode)
Чтобы проверить, как проект будет работать на реальном боевом сервере, необходимо собрать его билд. В папке `/frontend` выполните команды по очереди:
```bash
npm run build
npm run start
```
Команда `npm run start` запустит оптимизированную версию сайта. Убедитесь, что нет ошибок оптимизации картинок или гидратации (Hydration errors).
