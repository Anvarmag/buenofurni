# Архитектура проекта (Architecture) — BUENOFURNI

**О проекте:** Модульный сайт премиального бренда мебели, собираемый из независимых блоков.
**Технологический стек:** Современный Frontend (семантический HTML5, адаптивный CSS/SCSS, производительный Vanilla JS / легковесный фреймворк).
**Бизнес-логика:** Корзина отсутствует. Основной упор на формы лидогенерации (заявка на заказ, связь через WhatsApp/Telegram). Цены открыты.

---

## 1. Структура страниц (MVP Pages List)

Сайт состоит из следующих ключевых страниц:
*   **Главная страница (Home):** Витрина бренда, флагманские продукты, философия, призыв к действию.
*   **Каталог стульев (Catalog - Chairs):** Список доступных моделей с базовой фильтрацией и ценами.
*   **Страница продукта (Product Detail Page):** Детальное описание конкретной модели (фотогалерея, характеристики, материалы, форма заказа).
*   **Индивидуальный заказ (Custom Order):** Форма-бриф для заказа мебели по индивидуальным проектам (для HoReCa, дизайнеров, B2B).
*   **Материалы (Materials):** Описание используемых дерева, шпона, тканей с качественными фото фактур.
*   **Производство (Production):** Рассказ о процессе создания мебели, ценностях и крафтовом подходе бренда.
*   **Контакты (Contacts):** Адрес шоурума/мастерской, карта, реквизиты, ссылки на соцсети.

---

## 2. Маршрутизация (URL Structure)

ЧПУ (Человеко-понятные URL) базируются на латинице для лучшей совместимости, шеринга в мессенджерах и эстетичности ссылок.

*   **Главная:** `/`
*   **Каталог стульев:** `/catalog/chairs`
*   **Страница продукта:** `/product/{slug}` *(например, `/product/lounge-chair-m1`)*
*   **Индивидуальный заказ:** `/custom`
*   **Материалы:** `/materials`
*   **Производство:** `/production`
*   **Контакты:** `/contacts`

---

## 3. Глобальный лейаут (Global Layout)

Сквозные элементы интерфейса, присутствующие на всех страницах (кроме возможных landing-версий под узкие кампании).

*   **Header (Шапка сайта):**
    *   Поведение: *Sticky* (прилипает к верхнему краю экрана при скролле).
    *   Состояния: Прозрачный на первом экране (над Hero-блоком), становится сплошным при скролле для читаемости.
*   **Footer (Подвал):** 
    *   Навигация, контакты, ссылки на юридическую информацию (Политика конфиденциальности, Оферта).
*   **Shared CTA Widget:** 
    *   Плавающая кнопка (Floating Action Button) или закрепленный внизу мобильного экрана бар с быстрыми ссылками (WhatsApp, Заказать звонок).

## 4. Система блоков (Blocks System Rules)

Проект строится на компонентном подходе. Описание каждого смыслового блока будет храниться в директории `/blocks/` в виде Markdown-файлов.

### Правила документации блока (`/blocks/{block-name}.md`):
Каждый файл блока обязан содержать следующие разделы:
*   **Purpose:** Какую бизнес-задачу или пользовательскую потребность решает блок.
*   **Content:** Список текстовых, медийных и интерактивных полей блока.
*   **Layout:** Схема расположения элемента (Desktop / Tablet / Mobile).
*   **Responsive notes:** Специфические правила перестроения сетки на разных экранах.
*   **SEO notes:** Разметка заголовков (H2/H3/etc.), schema.org спецификации и alt-атрибуты.
*   **Accessibility notes (a11y):** Атрибуты ARIA, управление фокусом, роли элементов, контрастность.

### Соглашения об именовании (Naming Conventions):
*   **CSS-классы:** Методология *BEM* (Блок__Элемент--Модификатор). Пример: `.product-card`, `.product-card__title`, `.product-card--featured`.
*   **Ассеты (Images/Videos):** Описание по шаблону: `[block-name]-[element-name]-[descriptor].[ext]`. Пример: `hero-bg-wood.jpg`, `product-chair-m1-front.webp`.

## 5. Контентная модель (Content Model)

Структура данных для ключевых сущностей продукта и процессов оформления.

### Модель Продукта (Product Entity):
*   `Name` (Строка) — Название модели мебели.
*   `Slug` (Строка) — Уникальный URL-идентификатор для роутинга.
*   `Price` (Число) — Базовая цена изделия.
*   `Dimensions` (Объект) — ШхВхГ, высота посадки, вес (в мм и кг).
*   `MaterialsBase` (Массив) — Доступные типы основы каркаса (березовая фанера, шпон дуба, МДФ, эмаль).
*   `UpholsteryOptions` (Массив) — Варианты обивки (микровелюр, букле, экокожа + коды и названия цветов).
*   `Availability` (Enum) — Текущий статус готовности (`В наличии` / `Под заказ`).
*   `Gallery` (Массив URL) — Фотографии товара (в интерьере и изолированные студио-фотографии на бежевом фоне).

### Модель CTA-воронок (Lead Generation Flows):
*   **Заявка через форму (Request Form):** Сбор имени, телефона, и необязательного комментария. Отправка заявки на почту или в Telegram-бот/CRM. Обязательный чекбокс согласия с политикой конфиденциальности. Вызов окна подтверждения (Thank You state / Modal).
*   **Прямой контакт (WhatsApp/Telegram):** Кнопка с предварительно сгенерированным текстом для мессенджера *(например: "Здравствуйте! Хочу уточнить детали по стулу {ProductName}...")*.

## 6. Правила производительности (Performance Rules)

Сайт категории премиум-сегмента должен работать безупречно плавно и грузиться мгновенно.
*   **LCP (Largest Contentful Paint):** Целевое значение — менее 2.5 секунд (в т.ч. на 3G/4G сетях). Использовать обязательный `<link rel="preload">` для изображений и шрифтов первого экрана.
*   **Оптимизация изображений:** Абсолютный отказ от тяжелого растра без обработки. Конвертация изображений в **WebP** или **AVIF**.
*   **Lazy Loading:** Атрибут `loading="lazy"` для всех изображений и iframe-карт, находящихся ниже линии начального скролла.
*   **Видео-фоллбэки:** Все видеофайлы грузятся асинхронно без блокировки основного потока `(autoplay muted loop playsinline)`. Обязательно использование свойства `poster` с легким предпросмотром JPG/WebP на момент загрузки ролика.

## 7. Правила SEO и индексации (SEO Rules)

Архитектура предполагает готовность платформы к органическому трафику с момента запуска.
*   **Meta Templates:** Системная генерация тегов `<title>` и `<meta name="description">` для шаблонов каталога и товарных карточек.
*   **Social & OpenGraph:** Внедрение микроразметки `og:image`, `og:title`, `og:description` и Twitter Cards для привлекательного сниппета при шеринге витрины в соцсетях и мессенджерах.
*   **Структурированные данные (JSON-LD):** 
    *   `Schema.org/Product` для товаров (Название, цена, валюта, фото, наличие).
    *   `Schema.org/Organization` и `Schema.org/LocalBusiness` на странице контактов.
*   **Технические параметры:**
    *   Использование `<link rel="canonical">` на всех страницах (особенно товарах), чтобы избежать санкций за дубли контента.
    *   Генерация обновляемого `sitemap.xml`.
    *   Настроенный `robots.txt` с корректными директивами (закрытие системных папок скриптов и стилей от ботов, если необходимо).

## 8. Дорожная карта развития (Roadmap)

### Phase 1: MVP (Минимально жизнеспособный продукт)
*   Дизайн, верстка и внедрение базовых страниц по списку MVP (Home, Catalog, Product Page, Contacts, Materials, Production, Custom Order form).
*   Разработка ключевых модулей (Hero banner, Product Grid, Forms, Image-text blocks).
*   Подключение форм с уведомлениями на почту/Telegram.

### Phase 2: Enhancements (Глубокие улучшения и расширение)
*   **Motion Design & Immersiveness:** Внедрение продвинутых scroll-анимаций, тонкого параллакса элементов и полноценных видео-обложек с учетом оптимизации производительности.
*   **Product Experience:** 3D-просмотр или расширенные конфигураторы (визуальное переключение обивки).
*   **Advanced Catalog:** Сложная комплексная фильтрация в каталоге (разбор по стилям, ценовым категориям, коллекциям) с мгновенной подгрузкой результатов без перезагрузки всей вкладки браузера.
*   **Localization (Multi-language):** Рефакторинг и масштабирование верстки под англоязычную версию (International/B2B branch).
