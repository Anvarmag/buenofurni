# Блок: Final CTA (Финальный призыв к действию) — BUENOFURNI

## 1. Цель блока
Служить мощным конверсионным "якорем" в самом конце главной страницы (и других контентных страниц) непосредственно перед подвалом (Footer). Этот блок предназначен для пользователей, которые доскроллили до конца, но еще не приняли решение о покупке. Его задача — предложить персонализированную помощь и снять барьер "сложного выбора".

## 2. Контент и структура (DOM Blueprint)
*   `<section class="final-cta">`
    *   `<div class="cta-container">`
        *   **Заголовок (H2):** `<h2>Подберём ткань и отделку под ваш интерьер</h2>`
        *   **Текст (Paragraph):** `<p>Оставьте заявку — мы предложим варианты цвета, ножек и рассчитаем стоимость.</p>`
        *   **Кнопки (`<div class="cta-actions">`):**
            1.  `Primary CTA`: `<button class="btn btn--primary">Оставить заявку</button>`
            2.  `Secondary CTA`: `<a href="..." class="btn btn--secondary btn--whatsapp">Написать в WhatsApp</a>`

## 3. Визуальная идея (Design & Aesthetics)
*   **Акцентная панель:** Чтобы блок не сливался с остальным белым фоном сайта, он должен быть оформлен в виде крупной "плашки" (Card) или растягиваться на всю ширину экрана с фоном.
*   **Премиальный фон:** Использовать глубокий фирменный акцентный цвет (Dark Plum/Deep Wine) со светлым текстом, либо очень мягкий, еле заметный градиент/шум (Subtle grain), чтобы передать ощущение "дорогого" бутика, предлагающего персональный сервис.
*   **Типографика (Bold):** Заголовок `H2` должен быть крупным и выразительным (Editorial style), задавая тон уверенности и профессионализма.
*   **Пространство:** Вокруг текста и кнопок должно быть очень много "воздуха" (Large paddings), чтобы блок выглядел чисто и статусно.

## 4. Mobile-first адаптация
*   **Лейаут:** На мобильных устройствах элементы выстраиваются в строгий вертикальный ряд (Заголовок -> Текст -> Primary Button -> Secondary Button).
*   **Выравнивание:** Текст центрируется для лучшего визуального баланса.
*   **Full-width кнопки:** Кнопки "Оставить заявку" и "WhatsApp" обязаны растягиваться на 100% ширины контейнера (`width: 100%; display: block`), чтобы в них было невозможно промахнуться пальцем. Отступ между кнопками — не менее `16px`.
*   **Отступы контейнера:** По краям экрана (Padding left/right) должны оставаться комфортные безопасные зоны (Safe areas) по `16-20px`.

## 5. Логика триггеров (Trigger logic to request modal)
*   Клики по кнопке `"Оставить заявку"` должны перехватываться JavaScript-событием.
*   Функция открывает центральное окно заявки: `openModal({ source: 'final_cta_home' })`.
*   *(Опционально)* Если пользователь дошел до этого блока через UTM-метку (например, с рекламы WhatsApp), кнопку Primary можно динамически переключить на прямой переход в мессенджер для бесшовного UX.

## 6. SEO заметка (SEO Note)
*   Блок должен содержать семантический тег `<h2>`.
*   Текст абзаца содержит хорошие длинные (Long-tail) ключевые слова: `"подберём ткань"`, `"варианты цвета"`, `"рассчитаем стоимость"`, что дает дополнительный органический контекст странице.
*   Кнопка WhatsApp обязана быть тегом `<a>` с реальной ссылкой (напр. `https://wa.me/79991234567?text=Здравствуйте...`), чтобы поисковики могли связать домен сайта с контактным номером компании.
